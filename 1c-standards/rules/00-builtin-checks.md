# Встроенные проверки АПК (логика из Ядра)

Эти проверки реализованы непосредственно в коде ядра АПК и не зависят от внешних дополнений (EPF).

## Проверка структуры модуля

Ядро парсит модуль на блоки и проверяет их порядок.

### Типы блоков модуля
- **Комментарий** — строки, начинающиеся с `//`
- **ОписаниеПеременной** — строки `Перем ...;`
- **ЗаголовокПроцедуры** — `Процедура ИмяПроцедуры(...)`
- **ОкончаниеПроцедуры** — `КонецПроцедуры`
- **ЗаголовокФункции** — `Функция ИмяФункции(...)`
- **ОкончаниеФункции** — `КонецФункции`
- **Операторы** — всё остальное (включая директивы компиляции `&НаСервере` и т.д.)

### Что анализируется в каждом блоке
- Наличие комментария (документации) перед процедурой/функцией
- Наличие ключевого слова `Экспорт`
- Директива компиляции (`&НаКлиенте`, `&НаСервере`, `&НаСервереБезКонтекста`)
- Признак асинхронного метода

## Проверка документации параметров

### Формат описания параметра
```
// Параметры:
//   ИмяПараметра - Тип - описание параметра.
```

### Правила проверки (АПК:1297)
- Имя параметра должно предшествовать описанию типа
- Тип отделяется от имени символом ` - ` (пробел-дефис-пробел)
- Ссылка "см." указывает на отсутствие имени параметра
- Описание типов в скобках должно соответствовать формату: `ТипОбъекта.ИмяОбъекта`

## Проверка прав и ролей

### Запрещённые полномочия по типам объектов

| Тип объекта | Запрещённые права |
|---|---|
| Справочники, Документы, Задачи, Бизнес-процессы | Добавление, Изменение, ИнтерактивноеДобавление, Редактирование, ИнтерактивнаяПометкаУдаления |
| ПланОбмена | Добавление, Изменение, ИнтерактивноеДобавление, Редактирование, ИнтерактивнаяПометкаУдаления, ИнтерактивноеСнятиеПометкиУдаления, ИнтерактивноеУдалениеПомеченных |
| Регистры (сведений, накопления, бухгалтерии) | Изменение, Редактирование, УправлениеИтогами |

## Проверка орфографии

### Механизм
- Интеграция с Hunspell (внешний словарь)
- Собственный словарь верных слов (регистр `ВерныеСлова`)
- Разбиение связных слов CamelCase на отдельные слова для проверки

### Что проверяется
- Комментарии в коде
- Наименования объектов метаданных
- Тексты форм
- Содержимое макетов

### Проверка случайных латинских символов
Обнаруживает смешение кириллицы и латиницы в одном слове (например, буква "c" латинская вместо кириллической "с").

## Система исключений из проверки

### Формат исключений в комментариях кода
```bsl
// АПК:XXXX-выкл  — отключить проверку правила XXXX
// ... код ...
// АПК:XXXX-вкл   — включить проверку обратно
```

Английский вариант:
```bsl
// ACC:XXXX-off
// ... code ...
// ACC:XXXX-on
```

Где `XXXX` — код ошибки из справочника `ОбнаруживаемыеОшибки`.

## Определение типа конфигурации

### Демо-конфигурация
Определяется по наличию слова "ДЕМО" в наименовании конфигурации.

### Версия платформы
Извлекается из пути запуска в формате `8.X.XX.XXXX` (3 компонента без номера сборки).

### Версия БСП
Определяется наличие подсистемы "СтандартныеПодсистемы" и её версия.

## Платформенные методы (исключения из проверки)

### Методы таблиц значений
Эти методы не проверяются на стандарты именования:
Вставить, ВыбратьСтроку, Выгрузить, Добавить, Загрузить, ЗаполнитьЗначения, Индекс, Итог, Количество, Найти, НайтиСтроки, Очистить, Получить, Скопировать, Сортировать, Удалить

### Методы объектов метаданных
Выбрать, ВыбратьИерархически, НайтиПоКоду, НайтиПоНаименованию, НайтиПоРеквизиту, ПолучитьМакет, ПолучитьФорму, ПолучитьФормуВыбора

## Известные коды ошибок АПК

| Код | Описание |
|---|---|
| АПК:1297 | Параметры процедур/функций не задокументированы или описаны некорректно |
| АПК:1416 | Нарушение формата описания метода |
| АПК:1441 | Нарушение структуры комментария параметра |
